Purpose

The backend lets users log in, upload documents, ask questions, and receive answers based on their own files. It also saves chats so users can return later.

Full Workflow

1
User Signs Up Or Logs InThe backend creates or checks the account and gives the browser login keys.
2
User Starts A ChatA conversation is created and messages are saved under it.
3
User Uploads A DocumentThe file is checked, saved, and linked to the user or chat.
4
Backend Reads The DocumentThe document text is extracted and split into smaller useful pieces.
5
User Asks A QuestionThe backend finds useful document parts and sends them with the question to the AI.
6
Chat ContinuesFollow-up questions, history, exports, feedback, and regenerated answers are supported.

What Each Main Area Does

AreaSimple Purpose
AuthHandles signup, login, logout, and staying logged in.
UsersStores user account details and profile information.
ConversationsCreates and manages chat threads.
MessagesSaves user messages and assistant answers.
DocumentsUploads files and prepares their text for chat.
AI ServiceBuilds the question with useful context and asks the AI for an answer.

Simple Example

If a user uploads an employee policy document and asks, "How many leave days do I get?", the backend finds useful leave-policy text and sends it with the question to the AI. The AI then answers using that document text.
