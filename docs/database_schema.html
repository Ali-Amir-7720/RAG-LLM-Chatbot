How The Tables Connect

Users own conversations and sessions. Conversations hold messages. Documents can belong to users and can be attached to conversations. Documents are split into chunks, and chunks have searchable representations.

users

Stores account identity and login credentials.

AttributeTypeDetails
idUUIDPrimary key. Defaults to gen_random_uuid().
usernameVARCHAR(50)Required. Unique username.
emailVARCHAR(255)Required. Unique email.
password_hashTEXTRequired. Stores hashed password, not raw password.
created_atTIMESTAMPTZRequired. Defaults to now().

user_sessions

Stores refresh-token sessions for login validity, expiry, rotation, and logout.

AttributeTypeDetails
idUUIDPrimary key. Defaults to gen_random_uuid().
user_idUUIDRequired. References users(id). Deletes cascade.
refresh_tokenTEXTRequired. Unique token stored server-side.
deviceVARCHAR(255)Optional device/browser label.
ip_addressVARCHAR(100)Optional IP address string.
is_revokedBOOLEANRequired. Defaults to false.
expires_atTIMESTAMPTZRequired. Session is invalid after this time.
created_atTIMESTAMPTZRequired. Defaults to now().
updated_atTIMESTAMPTZRequired. Defaults to now().

AttributeTypeDetails

user_idUUIDRequired. References users(id). Deletes cascade.

updated_atTIMESTAMPTZRequired. Defaults to now().
